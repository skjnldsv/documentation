O actualizador automático

O actualizador automático asegura que sempre ten as últimas funcionalidades e correccións de erros para o seu cliente de sincronización de Nextcloud.

O actualizador automático Actualiza só en computadoras macOS e Windows; Os usuarios de Linux só precisan usar os seus xestores de paquetes normais. Non obstante, nos sistemas Linux o actualizador comprobará as actualizacións e avisará cando estea dispoñíbel unha nova versión.

Fluxo básico de traballo

As seguintes seccións describen como usar o actualizador automático en diferentes sistemas operativos.

Windows

O cliente de NextCloud comproba as actualizacións e descárgaas cando están dispoñíbeis. Pode ver o estado de actualización en Axustes > Xeral > Actualizacións no cliente de NextCloud.

Se hai unha actualización dispoñíbel e foi descargada correctamente, o cliente NextCloud inicia unha actualización silenciosa antes do seu seguinte inicio e após reiníciase. Se a actualización silenciosa falla, o cliente ofrece unha descarga manual.

Nota

Necesítanse privilexios administrativos para realizar a actualización.

macOS

O cliente de macOS ten un actualizador automático que emprega o marco de traballo Sparkle. Este actualizador automático está incluído mo paquete de aplicación do cliente e comproba se hai actualizacións ao iniciarse, notificándoo se hai unha actualización dispoñíbel. Isto presentará unha xanela emerxente que lle permitirá descargar e instalar automaticamente a última actualización do cliente premendo só unha vez.

Nas versións do cliente nas que non está empaquetado o actualizador automático baseado en Sparkle, aparecerá unha notificación, para premer nela, informando dunha actualización dispoñíbel. Ao premer na devandita notificación, abrirase a páxina de descarga da última versión do cliente no navegador web do sistema.

Coma noutros sistemas, pode ver o estado de actualización en Axustes > Xeral > Actualizacións no cliente NextCloud.

Linux

As distribucións de Linux fornecen as súas propias ferramentas de actualización, polo que os clientes de NextCloud que usan o sistema operativo Linux non realizan actualizacións por conta propia. O cliente informaralle (Axustes > Xeral > Actualizacións) cando estea dispoñíbel unha actualización.

Evitar as actualizacións automáticas

En contornos controlados, como empresas ou universidades, é posíbel que non queira activar o mecanismo de actualización automática, xa que interfire con ferramentas e directivas de despregadura controladas. Para abordar este caso, é posíbel desactivar completamente o actualizador automático. As seguintes seccións describen como desactivar o mecanismo de actualización automática para diferentes sistemas operativos.

Evitar as actualizacións automáticas en contornos Windows

Os usuarios poden desactivar as actualizacións automáticas engadindo esta liña á sección [General] dos seus ficheiros nextcloud.cfg:

skipUpdateCheck=true

Os administradores de Windows teñen máis opcións para evitar as actualizacións automáticas nos contornos de Windows mediante un dos dous métodos. O primeiro método permite aos usuarios anular o mecanismo de comprobación de actualización automática, mentres que o segundo método impide calquera anulación manual.

Para evitar actualizacións automáticas, mais permitir as anulacións manuais:

  1. Edite estas claves do rexistro:

    1. (Windows 32-bit) HKEY_LOCAL_MACHINE\Software\Nextcloud\Nextcloud

    2. (Windows 64-bit) HKEY_LOCAL_MACHINE\Software\Wow6432Node\Nextcloud\Nextcloud

  2. Engada a clave skipUpdateCheck (ou escriba DWORD).

  3. Especifique un valor de 1 á máquina.

Para anular manualmente esta clave, use o mesmo valor en HKEY_CURRENT_USER.

Para evitar actualizacións automáticas e non permitir anulacións manuais:

Nota

Este é o método preferido de controlar o comportamento do actualizador mediante as directivas de grupo.

  1. Edite esta clave do rexistro:

    HKEY_LOCAL_MACHINE\Software\Policies\Nextcloud GmbH\Nextcloud

  2. Engada a clave skipUpdateCheck (ou escriba DWORD).

  3. Especifique un valor de 1 á máquina.

Nota

Os clientes de marca teñen nomes de clave diferentes

Evitar as actualizacións automáticas en contornos Linux

Debido a que o cliente Linux non fornece unha funcionalidade de actualización automática, non é necesario retirar a comprobación de actualización automática. Non obstante, se quere desactivalo, edite o seu ficheiro de configuración do cliente de escritorio, $HOME/.config/Nextcloud/nextcloud.cfg. Engada esta liña á sección [General]:

skipUpdateCheck=true
