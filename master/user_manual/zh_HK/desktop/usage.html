使用同步客戶端

Nextcloud 桌面客戶端在背景運行，並在系統托盤（Windows、KDE）、選項單欄（macOS）或通知區域（Linux）顯示為圖標。

狀態指示器使用圖標顯示您同步的當前狀態。綠色圓圈配白色勾號表示您的同步是最新的，並且您已連接到 Nextcloud 伺服器。

藍色圖標配白色半圓表示正在進行同步。

黃色圖標配平行線表示您的同步已暫停。（最有可能是您暫停的。）

灰色圖標配三個白點表示您的同步客戶端已與 Nextcloud 伺服器失去連接。

當您看到黃色圓圈中的符號「！」時，這是信息圖標，您應該點擊它以查看其通知內容。

紅色圓圈配白色「x」表示配置錯誤，例如登錄或伺服器 URL 錯誤。

系統托盤圖標

右鍵單擊系統托盤圖標會打開一個選項單，以便快速訪問多個操作。

此選項單提供以下選項：

  • 打開主對話框

  • 暫停同步/恢復同步

  • 設定

  • 退出 Nextcloud，登出並關閉客戶端

左鍵單擊您的系統托盤圖標將打開桌面客戶端的主對話框。

主對話框顯示最近的活動、錯誤和伺服器通知。

單擊主對話框後，再單擊用戶的頭像，可以打開設置。

配置 Nextcloud 帳戶設置

窗口頂部是每個配置的同步帳戶的選項卡，還有兩個選項卡用於一般和網絡設置。在您的帳戶選項卡上，您擁有以下功能：

  • 連接狀態，顯示您連接的 Nextcloud 伺服器和您的 Nextcloud 用戶名。

  • 伺服器上已用空間和可用空間。

  • 當前同步狀態。

  • **添加資料夾同步連接**按鈕。

位於同步狀態欄右側的小按鈕（溢出選項單）提供其他選項：

  • 打開資料夾

  • 選擇要同步的內容（僅在您的檔案樹折疊時顯示，並展開檔案樹）

  • 暫停同步 / 恢復同步

  • 移除資料夾同步連接

  • 可用性（僅在啟用虛擬檔案支持時可用）

  • 啟用虛擬檔案支持/禁用虛擬檔案支持

**打開資料夾**會打開您的本地 Nextcloud 同步資料夾。

暫停同步**會暫停同步操作而不對您的帳戶進行任何更改。它會繼續更新檔案和資料夾列表，但不會下載或更新檔案。要停止所有同步活動，請使用 **移除資料夾同步連接。

Note

Nextcloud 不會保留目錄的 mtime（修改時間），但會更新檔案的 mtime。請參見 同步時資料夾日期錯誤 以獲取討論。

添加新帳戶

您可以在桌面同步客戶端中配置多個 Nextcloud 帳戶。只需在任何帳戶標籤上單擊 帳戶 > 添加新帳戶 按鈕以添加新帳戶，然後按照帳戶創建向導進行操作。新帳戶將作為新選項卡出現在設置對話框中，您可以隨時調整其設置。使用 帳戶 > 移除 來刪除帳戶。

檔案管理器覆蓋圖標

Nextcloud 同步客戶端提供覆蓋圖標，除了正常的檔案類型圖標外，還為您的系統檔案管理器（Windows 的資源管理器、Mac 的 Finder 和 Linux 的 Nautilus）顯示 Nextcloud 檔案的同步狀態。

覆蓋圖標類似於上述的系統托盤圖標。根據同步狀態和錯誤，它們在檔案和目錄上的行為不同。

單個檔案的覆蓋圖標表示其當前的同步狀態。如果該檔案與伺服器版本同步，則顯示綠色勾號。

如果該檔案被忽略同步，例如因為它在您的排除列表中，或因為它是符號鏈接，則顯示警告圖標。

如果出現同步錯誤，或該檔案被列入黑名單，則顯示引人注目的紅色 X。

如果該檔案等待同步或正在同步，則覆蓋圖標顯示藍色循環圖標。

當客戶端處於離線狀態時，將不顯示任何圖標，表示該資料夾當前不同步，並且沒有更改同步到伺服器。

已同步目錄的覆蓋圖標表示該目錄中檔案的狀態。如果有任何同步錯誤，則該目錄會標記為警告圖標。

如果目錄包含被標記為警告圖標的忽略檔案，則不會改變父目錄的狀態。

設置用戶狀態

如果您在 Nextcloud 伺服器上安裝了用戶狀態應用，則可以從桌面客戶端設置您的用戶狀態。為此，打開主對話框，然後單擊您的頭像，再單擊三個點。在打開的選項單中單擊 設置狀態。

在打開的對話框中，如果您單擊 在線、離開、勿擾 或 隱身，則可以設置您的在線狀態。您還可以使用下方的文本框設置自定義狀態消息，或選擇下面的預設狀態消息。如果單擊文本輸入框旁邊的表情符號按鈕，還可以設置自定義表情符號。最後，您可以設置用戶狀態何時應被清除。您可以通過單擊文本 清除狀態消息後 左側的按鈕來選擇清除用戶狀態的時間。

如果您對創建的狀態滿意，可以使用 設置狀態消息 按鈕啟用此狀態。如果您已設置狀態，可以通過單擊 清除狀態消息 按鈕來清除該狀態。

從桌面共享

Nextcloud 桌面同步客戶端與您的檔案管理器集成。macOS 使用 Finder，Windows 使用資源管理器。Linux 用戶必須根據使用的檔案管理器安裝附加包。例如，您可以使用 nautilus-nextcloud``（Ubuntu/Debian）、``dolphin-nextcloud``（Kubuntu）、``nemo-nextcloud 和 caja-nextcloud。您可以創建共享鏈接，並以與 Nextcloud 網頁界面相同的方式與內部 Nextcloud 用戶共享。

在檔案管理器中，單擊一個檔案，在上下文選項單中轉到 Nextcloud，然後單擊 共享選項 以調出共享對話框。

通過此對話框，您可以共享檔案。

常規窗口

常規窗口具有配置選項，例如 開機啟動時啟動、使用單色圖標 和 顯示桌面通知。在此您將找到 編輯被忽略檔案 按鈕，以啟動被忽略檔案編輯器，以及 在下載大於 [資料夾大小] 的資料夾之前請求確認。

使用網絡窗口

網絡設置窗口使您可以定義網絡代理設置，並限制下載和上傳帶寬。

使用被忽略檔案編輯器

您可能有一些近端檔案或目錄不希望備份並存儲在伺服器上。要識別並排除這些檔案或目錄，您可以使用 *被忽略檔案編輯器*（常規選項卡）。

為方便起見，編輯器預填充了一個典型忽略模式的默認列表。這些模式包含在一個系統檔案中（通常是 sync-exclude.lst），位於 Nextcloud 客戶端應用程序目錄中。您不能直接從編輯器修改這些預填充的模式。然而，如果需要，您可以將鼠標懸停在列表中的任何模式上，以顯示與該模式相關的路徑和檔案名，找到該檔案並編輯 sync-exclude.lst 檔案。

Note

修改全局排除定義檔案可能會使客戶端無法使用或導致不期望的行為。

編輯器中的每一行包含一個忽略模式字符串。在創建自定義模式時，除了能夠使用正常字符來定義忽略模式外，您還可以使用通配符字符來匹配值。例如，您可以使用星號（*）來表示任意數量的字符，或使用問號（?）來表示單個字符。

以斜杠字符（/）結尾的模式僅適用於被檢查路徑的目錄組件。

Note

編輯器目前不會對自定義條目進行語法正確性的驗證，因此您不會看到任何錯誤語法的警告。如果您的同步未按預期工作，請檢查您的語法。

列表中的每個模式字符串後面都有一個復選框。當復選框中包含勾號時，除了忽略與模式匹配的檔案或目錄組件外，任何匹配的檔案也被視為「瞬態元數據」，並由客戶端刪除。

除了排除使用此列表中定義的模式的檔案和目錄外：

  • Nextcloud 客戶端始終排除包含無法同步到其他檔案系統的字符的檔案。

  • 在同步過程中，導致個別錯誤三次的檔案將被刪除。然而，客戶端提供了對產生錯誤的檔案再重試三次同步的選項。
